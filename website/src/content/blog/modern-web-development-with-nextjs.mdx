# Build for change, not just launch day

Next.js gives you strong defaults, but architecture decisions still decide long-term velocity.

## What we optimize for

1. Small client bundles
2. Clear server/client boundaries
3. Predictable data loading paths

```tsx
export default async function Page() {
  const data = await getDashboardData();
  return <Dashboard data={data} />;
}
```

When in doubt, push rendering to the server and make client components truly interactive islands.
